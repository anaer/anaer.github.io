# Velocity在Java中的使用

``` java
// 初始化模板引擎
ve = new VelocityEngine();

// 设置配置信息 可以将配置信息写到配置文件中
Properties properties = new Properties();
String basePath = "src\\main\\resources\\velocity\\work";// 这里需要这样写路径！！！
// 设置模板的路径
properties.setProperty(Velocity.FILE_RESOURCE_LOADER_PATH, basePath);

// 设置模板文件编码
properties.setProperty(Velocity.INPUT_ENCODING, "utf-8");
// 设置输出流编码
properties.setProperty(Velocity.OUTPUT_ENCODING, "utf-8");

// Log4j 插件可能会冲突， 详情 见velocity.wiki
ve.setProperty(VelocityEngine.RUNTIME_LOG_LOGSYSTEM_CLASS, "org.apache.velocity.runtime.log.NullLogChute");
// 初始化velocity 让配置生效
ve.init(properties);

//ve.init(System.getProperty("user.dir") + "\\src\\main\\resources\\" + "velocity.properties");
```

# Velocity模板文件定义*.vm
  1. 宏的定义

	```vm
	#macro(宏的名称 $参数1 $参数2 ...)
      语句体(即函数体)
	#end
	```

  2. 宏的调用

	```vm
	#宏的名称($参数1 $参数2 ...)
	```

  举例:

  1. 简单宏

  定义:

    #macro( d )
    <tr><td></td><tr>
    #end

  使用:

    #d()

  2. 宏参数

  定义:

    #macro( d $name)
        <tr><td>$name</td></tr>
    #end

  使用:

    #d("name1")

# velocity遇到的问题及分析

使用情况：

        使用了webmvc框架，框架里集成了velocity模板引擎，但是在使用时没有配置velocity.properties日志配置文件。

现象：

        使用jenkins发布后应用一直不能正常启动（tomcat端口已经监听），具体错误信息如下：

错误信息：

        jenkins发布后的错误信息：

问题原因:

        velocity的日志框架导致（velocity是使用自己封装的日志框架记录日志的），velocity在初始化Logger时，如果没有读取到配置文件，则会使用默认的velocity.log做为文件输出路径，源代码里使用了 File file = new File("velocity.log") 代码片段,这样创建的文件目录是在启动jvm进程的用户目录下（也就是user.dir属性）。

解决办法：

        1、添加一个velocity.properties日志配置文件
        2、修改velocity的代码，把日志转接到应用里使用的日志框架上来。
        解决的方式是直接关闭Velocity日志。
        3/在调用 Velocity.init(); 之前
        设置：
        Velocity.setProperty(VelocityEngine.RUNTIME_LOG_LOGSYSTEM_CLASS, "org.apache.velocity.runtime.log.NullLogChute");
