#### Error updating database.  Cause: com.mysql.jdbc.PacketTooBigException: Packet for query is too large (1378267 > 1048576). You can change this value on the server by setting the max_allowed_packet' variable.

    MySQL的一个系统参数：max_allowed_packet，其默认值为1048576(1M)，
    修改my.ini/my.cnf配置
    [mysqld]
    max_allowed_packet = 1M （ 将1M更改为4M(如果没有这行内容，增加一行))，

#### Every derived table must have its own alias
    子查询的结果必须有一个别名

#### ERROR 2002 (HY000): Can't connect to local MySQL server through socket '/var/run/mysql.sock' (2)
    [原文链接](http://www.web-build.com.au/zh-tw/cygwin-mysql/)

    ```
    當我們編譯安裝完 MySQL Client，打開 Windows 系統中的 MySQL Server，然後鍵入mysql -u root -p是發現報錯:

    error 2002 (hy000) can’t connect to local mysql server through socket ‘/tmp/mysql.sock’ (2)

    默 認的情況下，不帶 -h 參數或者使用 -h localhost，MySQL 都會使用 Unix socket file 連接伺服器，即使你在命令中指定了埠也會被忽略的，所以肯定連接不上的，提示找不到 /tmp/mysql.sock。使用 IP 或者主機名後，MySQL 就會使用 TCP/IP 模式連接伺服器的 3306 埠，這樣就什麼沒問題了。

    為了方便，在配置文件中強制客戶端使用 TCP/IP 連接模式，複製 mysql-5.5.0-m2/support-files 中的配置文件樣板到 /etc/my.cnf：
    $ cp support-files/my-medium.cnf /etc/my.cnf

    修改my.cnf文件如下：
    # The following options will be passed to all MySQL clients
    [client]
    #password   = your_password
    port        = 3306
    socket      = /tmp/mysql.sock
    protocol    = TCP

    ## 指定客戶端連接的默認編碼，注意是 utf8，不是 utf-8
    ## 可根據需要自行修改
    default-character-set = utf8

    之後我們再次鍵入mysql -u root -p的時候就會提示我們輸入密碼，mysql客戶端成功連接到伺服器。
    ```

#### 登录MySQL时，报错ERROR 1045 (28000): Access denied for user root@localhost (using password: NO)

  ```
  1、停用 mysql 服务：
    # /etc/rc.d/init.d/mysqld stop 

  2 、输入命令： 
    # mysqld_safe --user=mysql --skip-grant-tables --skip-networking & 

  3 、登入数据库： 
    # mysql -u root mysql 

  4 、 
     mysql> use mysql;

  5 、
     mysql> UPDATE user SET Password=PASSWORD('newpassword')where USER='newuser'; 
     mysql>FLUSH PRIVILEGES; 
     mysql> quit 

    # /etc/init.d/mysql restart 
    # mysql -u newuser –p 
	Enter password:newpassword 

	mysql>< 登录成功 >  	
  ```

  * 命令行查询数据 中文乱码
  
	```mysql
	set names gbk;
	```
	
  6、 还有情况是机子上存在多个mysql服务端，需要指定主机和端口
  
  ```sql
  mysql -u root -h 127.0.0.1 -P3306 database -ppassword
  ```
